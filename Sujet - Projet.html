AP1 Party Game¶

Présentation¶

Mario Party est un jeu de plateau où les joueurs avancent chacun leur tour, dans le but, principalement, d’acheter le plus d’étoiles possible.

Nous proposons, dans ce projet, une version simplifiée du jeu Mario Party.

Le nombre de joueurs pour ce projet pourra varier entre 1 à 4 joueurs.

Les étapes du projet¶

Le projet se décompose en différentes étapes :

  • Définir les structures de données appropriées nécessaires pour le plateau et les joueurs
  • Prendre en main le module graphique proposé (à télécharger dans la section “Interface graphique”)
  • Initialiser le jeu
  • Effectuer les différents tours de jeu
  • Classer les joueurs en fin de partie
  • Exporter les résultats du jeu dans un fichier

Le programme consistera à réaliser une boucle de tours de jeu dont le nombre de tours sera défini par l’utilisateur.

Structures de données¶

Le plateau¶

Le plateau de jeu est déterminé par une liste d’entiers. Ainsi, une position sur le plateau correspond à l’indice d’un élément de la liste (la position 0 par exemple est représentée par le premier élément de la liste).

Les valeurs de la liste sont des entiers compris entre 1 et 3 représentant les différentes cases du plateau : - 1 représente une case verte ; - 2 représente une case rouge ; - 3 représente une case bleue.

Étant donné que le plateau de jeu est carré, vous veillerez à ce que le nombre de cases soit cohérent.

Par exemple, on définit un plateau dont chaque côté possède 4 cases, possédant 7 cases vertes, 3 cases rouge et 2 cases bleues avec la liste :

[2,1,1,1,2,3,1,1,1,1,2,3]

Les joueurs¶

Afin de représenter les joueurs, vous utiliserez une liste de dictionnaires, dont chaque dictionnaire contient les informations nécessaires pour un joueur.

Les informations minimales pour chacun des joueurs sont :

  • position : sa position sur le plateau ;
  • pieces : le nombre de pièces que possède le joueur ;
  • etoiles : le nombre d’étoiles qu’il possède ;
  • objets : la liste des objets possédés par le joueur.

Vous pourrez, bien sûr, ajouter, selon vos besoins, d’autres informations pour chacun des joueurs.

L’étoile¶

L’étoile est uniquement définie par une position sur le plateau.

Interface graphique¶

Un code permettant d’obtenir une interface graphique pour ce jeu est disponible en téléchargeant le fichier interface.zip. Une fois défini l’ensemble des structures de données, elles peuvent être directement visualisées en utilisant les différentes fonctions implémentées dans ce module. Les tours de jeu sont alors effectués en ligne de commande et l’évolution des joueurs sur le plateau peut être visualisée à l’aide des fonctions :

cree_plateau(m):
"""
Crée le plateau sur le screen. Le plateau est décrit par une liste d'entiers.
:param m: (list) liste d'entiers correspondant au plateau à dessiner.
Un 1 dans la liste correspond à une case verte, un 2 correspond à une case rouge et un 3 à une case bleue.
:return: (None)
:CU: len(m) doit être un multiple de 4
"""

cree_perso(joueurs,m):
"""
Ajoute les joueurs sur le plateau.
:param joueurs: (list) liste de dictionnaires.
:param m: (list) liste d'entiers correspondant au plateau.
:return: (None)
:CU: chaque dictionnaire doit contenir une clé "position" correspondant à une position sur le plateau m.
Pour chaque dictionnaire de joueurs, -1 < j["position"] < len(m)
"""

place_etoile(ind,m):
"""
Place l'étoile sur le plateau.
:param ind: (int) indice de la case où doit être positionnée l'étoile.
:param m: (list) liste d'entiers correspondant au plateau.
:return: (None)
:CU: -1 < ind < len(m)
"""

bouge_perso(joueurs,m):
"""
Modifie la position d'un personnage sur le plateau.
:param joueurs: (list) liste de dictionnaires.
:param m: (list) liste d'entiers correspondant au plateau.
:return: (None)
:CU: chaque dictionnaire doit contenir une clé "position" correspondant à une position sur le plateau m.
Pour chaque dictionnaire de joueurs, -1 < j["position"] < len(m)
"""

Règles du jeu¶

Avant la partie¶

  • Il faut définir le nombre de joueurs pour la partie.
  • Chaque joueur choisit un pseudo.
  • Les joueurs définissent le nombre de tours qu’ils souhaitent jouer.

Au début de la partie¶

  • Chaque joueur reçoit 5 pièces.
  • Le plateau est initialisé.
  • L’étoile est placée aléatoirement sur une case du plateau.
  • Tous les joueurs sont placés sur la première case du plateau.

Objets¶

Dans cette version, les joueurs peuvent gagner des objets. Les objets peuvent permettre :

  • de voler 5 pièces à un adversaire au choix ;
  • de voler une étoile à un adversaire au choix ;
  • d’ajouter 3 à son dé ;
  • de retirer 3 au dé d’un adversaire au choix.

Tour de jeu¶

Pendant un tour de jeu, les joueurs ont chacun leur tour la main.

Pendant qu’un joueur a la main, il peut, au début de son tour, utiliser un objet parmi ceux qu’il a en sa possession.

Chaque joueur lance un dé et avance sur le plateau en fonction du score effectué par le dé et du bonus ou malus s’il y a (suite à l’utilisation d’un objet). Si le joueur passe par la case où est située l’étoile et si il a suffisemment de pièces, il lui est proposé de l’acheter. Si le joueur accepte, le nombre de pièces du joueur diminue alors du prix de l’étoile et l’étoile est replacée aléatoirement sur le plateau.

Après s’être déplacé, le joueur peut être placé sur différents types de cases :

  • une verte lui apporte 3 pièces ;
  • une rouge lui fait perdre 3 pièces ;
  • une bleue lui fait gagner un objet aléatoirement.

A la fin du jeu¶

Les joueurs sont classés en fonction du nombre d’étoiles gagnées et du nombre de pièces. Un premier classement est fait selon le nombre d’étoiles. Lorsque deux joueurs ont le même nombre d’étoiles, ils sont départagés en fonction du nombre de pièces. Le gagnant est alors annoncé et un fichier contenant les scores de chaque joueur ainsi que les statistiques de jeux (nombre de cases parcourues par chaque joueurs, nombre d’objets collectés par joueur…) est créé.

Améliorations possibles¶

  • Proposer des mini-jeux à la fin de chaque tour. Par exemple : chaque joueur lance un dé, le joueur qui fait le score le plus élevé gagne des pièces. Faites jouer votre imagination pour créer les mini jeux.
  • Imaginer d’autres objets.
  • Imaginer des étoiles bonus, reçues en fonction de différentes statistiques du jeu (le joueur ayant reçu le plus d’objets, le joueur ayant parcouru le plus de cases, …)
